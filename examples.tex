As a motivating example, we will show how our features for stepping
through programs can help clarify the execution of a map function.

A Hazel implementation of a map function over lists of integers is
given in \autoref{fig:hazel-map}. The full execution trace is shown in
\autoref{fig:full-trace}. It shows each of the small-step transitions
that are taken when evaluating this program.

In some circumstances, seeing the full length of the execution trace
can be distracting. We may, for example, only want to focus on the
recursive calls. The Hazel stepper provides some filter primitives
that specify steps to hide. To more clearly show the
intuition behind the \verb|map| function, we can hide all steps except 
those that show the application of the \verb|map| function, by inserting a
hide filter and a pause filter as shown in
\autoref{fig:hide-pause-map}.

\begin{figure}
  \centering
  \begin{subfigure}[T]{0.40\linewidth}
    \frame{\includegraphics[scale=0.4]{images/eg21.png}}
    \Description{Map example annotated with debug eval and debug stop filters to focus on recursive calls.}
    \caption{The code from \autoref{fig:hazel-map} modified with extra hide and pause statements.}
    \label{fig:hide-pause-map}
  \end{subfigure}

Each filter has a pattern that triggers the filter. The \verb|$e|
pattern matches any expression, and the \verb|$v| pattern matches any
fully-evaluated expression. \verb|map($v, $v)| is a pattern that
matches any map expression applied to two fully-evaluated
expressions. More details on the patterns are given in the 
pattern matching judgment (\fbox{\(\Matches{p}{e}\)}).

The construct \verb|hide| hides a step that matches the pattern, and thus
\verb|hide $e| hides all steps. The construct \verb|stop| shows a
step that matches the pattern. Since it comes later, 
\verb|debug stop(map($v, $v))| overrides the expression \verb|hide| to show
any steps taken from an expression that contains a \verb|map| applied
to two values.

The \verb|hide| and \verb|pause| constructs each hide or show only one
step. We also provide two more constructs, \verb|eval| and
\verb|debug|, that hide or show all the evaluation steps for a
sub-expression that matches the given filter.

For example, in \autoref{fig:debug-map} we have added a debug
filter that shows the full trace of each application of \verb|f|. We
can see in \autoref{fig:debug-trace} that this not only
shows steps that apply \verb|f|, but also multiplications that
take place to evaluate \verb|f(x)|.

\begin{figure}
  \centering
  \begin{subfigure}{0.40\linewidth}
    \frame{\includegraphics[scale=0.35]{images/eg31.png}}
    \Description{Map example with eval and step filters, including debug mode for square.}
    \caption{The code from \autoref{fig:hazel-map} modified with extra hide, pause, and debug statements.}
    \label{fig:debug-map}
  \end{subfigure}
  \hfill
  \begin{subfigure}{0.57\linewidth}
    \frame{\includegraphics[scale=0.28]{images/eg32.png}}
    \Description{Filtered trace showing recursive map steps and explicit square evaluations on list elements.}
    \caption{Stepper output showing only the recursive calls and the steps taken to evaluate squares.}
    \label{fig:debug-trace}
  \end{subfigure}
  \caption{A filtered stepper that demonstrates debug mode.}
\end{figure}

Note that this \verb|debug| statement overrides the \verb|pause| and
\verb|eval| statements because of the dynamic scoping of filters, which
is achieved by instrumenting the expression at every step and selecting the
closest filter to the part where the next step occurs (see
Step~\ref{num:simple_example_decompose} in
Section~\ref{sec:filter_example} for a more detailed description of
this).
